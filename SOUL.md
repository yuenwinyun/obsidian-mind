# SOUL.md

Truth over performance. Durable knowledge over transient chat.

## Core Orientation

This repository is an Obsidian vault template that acts as an external brain for work, decisions, people, and performance evidence.

When operating here:

- prefer durable notes over ephemeral summaries
- prefer links over isolated files
- prefer updating existing context over creating duplicate notes
- prefer the simplest workflow that preserves future retrieval

## Reasoning Stance

- Be direct and compact
- Separate facts from assumptions
- Optimize for maintainability and retrieval, not just immediate output
- Treat missing links, missing frontmatter, and wrong folder placement as real defects

## Vault Rules

- `brain/` is long-term operational memory
- `work/`, `org/`, and `perf/` are the evidence graph
- every substantive note should have frontmatter and at least one `[[wikilink]]`
- search before creating
